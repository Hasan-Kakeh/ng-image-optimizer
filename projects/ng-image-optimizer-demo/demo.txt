What v1.0.0 changes:
The library has been completely rewritten with a dual-engine architecture, fully tree-shakable, split into two isolated operational modes:
SSR / Dynamic Mode

provideImageOptimizerLoader({ routePrefix: '/_ng/image' }) on the app side
imageOptimizerHandler(browserDistFolder, options) Express middleware on the server side
Uses Sharp to dynamically compress, reformat (WebP/AVIF), and resize on request
Best for SSR apps, dynamic platforms, user-uploaded media

AOT / Build Mode (entirely new in v1.0.0)

provideAotImageLoader({ format: 'webp', quality: 90 }) on the app side
ng-image-optimizer-aot CLI script run as a postbuild step
Pre-compiles all images into all responsive breakpoint sizes at build time
Output follows a deterministic naming convention: assets/hero.jpg → assets/hero.750q90.webp
Zero runtime overhead — best for SSG, Jamstack, GitHub Pages, Vercel, Netlify

Shared by both engines:

Both override Angular's IMAGE_CONFIG token with a full breakpoint matrix: imageSizes: [16, 32, 48, 64, 96, 128, 256, 384] and deviceSizes: [640, 750, 828, 1080, 1200, 1920, 2048]
Both enforce CLS prevention by preserving image dimensions through the loader resolution chain
